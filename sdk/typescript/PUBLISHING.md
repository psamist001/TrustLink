# TypeScript SDK Publication Guide\n\nThis document explains how the TrustLink TypeScript SDK is automatically published to npm.\n\n## Overview\n\nThe TypeScript SDK is published to npm as `@trustlink/sdk` using GitHub Actions. Publication happens automatically when:\n\n1. **New releases are created** - The SDK version matches the git tag\n2. **Manual workflow dispatch** - For testing or emergency releases\n\n## Setup Requirements\n\n### 1. npm Token\n\nThe repository needs an `NPM_TOKEN` secret configured:\n\n1. Go to [npm.com](https://www.npmjs.com) and log in\n2. Navigate to **Access Tokens** in your account settings\n3. Create a new **Automation** token with **Publish** permissions\n4. Add the token as `NPM_TOKEN` in GitHub repository secrets\n\n### 2. Package Configuration\n\nThe `sdk/typescript/package.json` includes:\n\n```json\n{\n  \"name\": \"@trustlink/sdk\",\n  \"publishConfig\": {\n    \"access\": \"public\"\n  },\n  \"files\": [\n    \"dist\",\n    \"README.md\",\n    \"LICENSE\"\n  ]\n}\n```\n\n## Publication Workflows\n\n### Automatic Publication (Recommended)\n\n**Trigger:** Creating a new GitHub release\n\n1. Create a git tag: `git tag v1.2.3`\n2. Push the tag: `git push origin v1.2.3`\n3. Create a GitHub release from the tag\n4. The workflow automatically:\n   - Extracts version from tag (`v1.2.3` тЖТ `1.2.3`)\n   - Updates `package.json` version\n   - Builds the TypeScript code\n   - Publishes to npm with `latest` tag\n\n### Manual Publication\n\n**Trigger:** Workflow dispatch in GitHub Actions\n\n1. Go to **Actions** тЖТ **Publish TypeScript SDK**\n2. Click **Run workflow**\n3. Specify:\n   - **Version**: `1.2.3` (without `v` prefix)\n   - **Tag**: `latest`, `beta`, or `alpha`\n\n## Build Process\n\nThe publication workflow:\n\n1. **Checkout** - Gets the latest code\n2. **Setup Node.js** - Installs Node.js 20 with npm cache\n3. **Install dependencies** - Runs `npm ci`\n4. **Update version** - Sets version in `package.json`\n5. **Build** - Runs `npm run build` (TypeScript compilation)\n6. **Type check** - Verifies TypeScript types\n7. **Verify build** - Lists output files and package contents\n8. **Check existing version** - Prevents duplicate publications\n9. **Publish** - Uploads to npm with provenance\n\n## File Structure\n\n```\nsdk/typescript/\nтФЬтФАтФА src/                 # TypeScript source files\nтФВ   тФЬтФАтФА client.ts       # Main TrustLinkClient class\nтФВ   тФЬтФАтФА types.ts        # Type definitions\nтФВ   тФФтФАтФА index.ts        # Public API exports\nтФЬтФАтФА dist/               # Built JavaScript (generated)\nтФВ   тФЬтФАтФА index.js\nтФВ   тФЬтФАтФА index.d.ts\nтФВ   тФФтФАтФА ...\nтФЬтФАтФА package.json        # npm package configuration\nтФЬтФАтФА tsconfig.json       # TypeScript compiler config\nтФЬтФАтФА .npmignore         # Files to exclude from npm\nтФФтФАтФА README.md          # Package documentation\n```\n\n## Published Package Contents\n\nThe npm package includes only:\n\n- `dist/` - Compiled JavaScript and type definitions\n- `README.md` - Package documentation\n- `LICENSE` - MIT license file\n- `package.json` - Package metadata\n\nExcluded files (via `.npmignore`):\n- `src/` - TypeScript source\n- `tsconfig.json` - Build configuration\n- Development and test files\n\n## Version Management\n\n### Semantic Versioning\n\nThe SDK follows [semver](https://semver.org/):\n\n- **Major** (`1.0.0` тЖТ `2.0.0`): Breaking changes\n- **Minor** (`1.0.0` тЖТ `1.1.0`): New features, backward compatible\n- **Patch** (`1.0.0` тЖТ `1.0.1`): Bug fixes, backward compatible\n\n### npm Tags\n\n- `latest` - Stable releases (default)\n- `beta` - Pre-release versions\n- `alpha` - Early development versions\n\n```bash\n# Install specific versions\nnpm install @trustlink/sdk@latest\nnpm install @trustlink/sdk@beta\nnpm install @trustlink/sdk@1.2.3\n```\n\n## Troubleshooting\n\n### Publication Fails\n\n**Error: \"Version already exists\"**\n- The version is already published to npm\n- Increment the version number and try again\n\n**Error: \"Invalid npm token\"**\n- Check that `NPM_TOKEN` secret is set correctly\n- Verify the token has publish permissions\n- Regenerate the token if needed\n\n**Error: \"Build failed\"**\n- Check TypeScript compilation errors\n- Verify all dependencies are correctly specified\n- Run `npm run build` locally to debug\n\n### Local Testing\n\n```bash\ncd sdk/typescript\n\n# Install dependencies\nnpm install\n\n# Build the package\nnpm run build\n\n# Test the build\nnpm pack --dry-run\n\n# Create a local package for testing\nnpm pack\n# This creates trustlink-sdk-1.2.3.tgz\n\n# Test in another project\ncd /path/to/test-project\nnpm install /path/to/trustlink-sdk-1.2.3.tgz\n```\n\n### Verify Publication\n\nAfter publication, verify the package:\n\n```bash\n# Check package info\nnpm info @trustlink/sdk\n\n# Install and test\nnpm install @trustlink/sdk\nnode -e \"console.log(require('@trustlink/sdk'))\"\n```\n\n## Security\n\n### npm Provenance\n\nThe workflow uses `--provenance` flag to:\n- Link the package to its source code\n- Provide cryptographic proof of origin\n- Enable supply chain security verification\n\n### Access Control\n\n- Only repository maintainers can trigger manual workflows\n- npm token is stored as an encrypted secret\n- Publication requires successful build and tests\n\n## Monitoring\n\n### GitHub Actions\n\n- View workflow runs in **Actions** tab\n- Check job summaries for publication details\n- Monitor for failed publications\n\n### npm Package\n\n- Monitor download statistics\n- Check for security vulnerabilities\n- Review user feedback and issues\n\n## Emergency Procedures\n\n### Unpublish a Version\n\n```bash\n# Unpublish within 24 hours (npm policy)\nnpm unpublish @trustlink/sdk@1.2.3\n\n# Deprecate a version (recommended)\nnpm deprecate @trustlink/sdk@1.2.3 \"This version has critical bugs\"\n```\n\n### Hotfix Release\n\n1. Create a hotfix branch from the problematic release\n2. Apply the minimal fix\n3. Increment the patch version\n4. Create a new release\n5. The workflow will automatically publish the fix\n\n## Related Documentation\n\n- [npm Publishing Guide](https://docs.npmjs.com/packages-and-modules/contributing-packages-to-the-registry)\n- [GitHub Actions Documentation](https://docs.github.com/en/actions)\n- [Semantic Versioning](https://semver.org/)\n- [TypeScript Handbook](https://www.typescriptlang.org/docs/)\n
